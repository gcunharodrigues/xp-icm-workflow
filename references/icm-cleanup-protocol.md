# ICM Cleanup Protocol — last active exit A/C (v3.7.2)

> **Version:** v3.7.2
> **Skill:** `xp-icm-workflow`
> **Triggered by:** stage 08 exits A/C, opt-in with human confirmation

## Purpose

Revert the ICM-ephemeral structure (workspace branches, `.icm-main/` worktree, subagent worktrees) back to the natural post-project state. After exit A (close last) or C (spawn last), the human typically wants:

1. `<project_root>/` (root) to reflect product code, not workspace branch state.
2. No parallel worktrees consuming disk.
3. No stale branches accumulating.

Without cleanup, the human sees:
- Root in `workspace/<NNN>-<slug>` branch (ephemeral state).
- `.icm-main/` parallel worktree duplicated in the same root.
- Orphan subagent worktrees in `.git/worktrees/<task-slug>/`.
- Workspace branch deletable but not deleted.

## Design decisions

### D1 — Cleanup is opt-in with confirmation

Default: human sees menu `[y/n/dry-run]` post-`/init` on exit A/C. Reason: cleanup is destructive (deletes branch + worktrees). Automatic without confirm risks erasing uncommitted work if human forgot to stash.

Rejected alternative: automatic on tier=experimental. Complexity > value — universal opt-in is more predictable.

### D2 — Implementation in a dedicated Python script

`scripts/icm-cleanup.py` instead of inline in the template. Reasons:
- Complex logic (pre-checks, order matters, `--force` fallback, dry-run).
- Deterministically testable (`tests/unit/test_icm_cleanup.py`).
- Reusable outside the template (recovery wizard, manual human use).

### D3 — Safety pre-checks

Abort BEFORE any mutation if:
- `project_root` is not a git repo.
- Workspace branch tree is dirty (uncommitted changes) — unless `--force`.
- `.icm-main/` worktree is dirty — unless `--force`.

`--force` flag exists but is destructive and marked dangerous in the help.

### D4 — Subagent worktrees from stage 04

Worktrees created via `Agent(isolation: "worktree")` in wave execution. Typical path outside `<project_root>` (generated by the Claude Code harness). Cleanup detects via `git worktree list --porcelain`, filters paths that are NOT `<project_root>` or `.icm-main/`. Removes with `--force` (assuming branch was already merged or discarded).

`git worktree prune` at the end cleans stale refs in `.git/worktrees/`.

### D5 — Exit A vs C vs B

| Exit | Cleanup offered? | Reason |
|---|---|---|
| **A** (close) | Yes, if exit 2 (last active) | Workspace closed, no follow-up — natural cleanup |
| **C** (spawn) | Yes, if exit 2 (last active) | New workspace will be bootstrapped in a separate session — bootstrap re-creates `.icm-main/` if absent |
| **B** (restart) | NEVER | Workspace still active, cleanup would destroy state in use |

In multi-workspace with remaining active workspaces after exit A/C (exit `0`), cleanup is also skipped — other workspaces depend on `.icm-main/`.

## Algorithm

Sequence (order matters):

```
1. Pre-checks (abort if they fail)
   - project_root is a git repo
   - workspace branch tree clean
   - .icm-main/ tree clean

2. Orphan subagent worktrees
   for wt in git worktree list (≠ project_root, ≠ .icm-main):
       git worktree remove <wt> --force

3. Remove .icm-main worktree (releases base_branch)
   git worktree remove .icm-main
   (fallback: --force if failed)

4. Checkout base_branch in project_root
   git checkout main
   (now possible because .icm-main exited in step 3)

5. Delete workspace branch
   git branch -D workspace/<NNN-slug>

6. Final prune
   git worktree prune
```

**Critical order:** `.icm-main/` MUST exit before checkout in project_root, otherwise git rejects "branch already checked out in another worktree".

## CLI

```bash
python scripts/icm-cleanup.py \
    --project-root <path> \
    --workspace <NNN-slug> \
    --base-branch main \
    [--dry-run]   # prints commands without executing
    [--force]     # ignores uncommitted changes (dangerous)
```

Exit codes:
- `0` = success (or dry-run complete)
- `1` = abort due to pre-check
- `2` = git command execution error

## Edge cases

| Edge case | Behavior |
|---|---|
| `.icm-main/` absent (legacy pre-v3.4.0) | Skip step 3, continue normally |
| Workspace branch already deleted | Warning + skip step 5 |
| Workspace branch has unmerged commits (exit C — code could be lost) | Pre-check passes (commits exist), visible warning in post-cleanup print |
| Project_root already on base_branch | Skip step 4, continue |
| Subagent worktrees still have valid branches | `--force` on `worktree remove` ignores; branch remains in repo (human may `git branch -D` manually) |
| Concurrent multi-workspace (human opened 2 sessions) | Cleanup only runs when `--exit-2-if-last-active` returned 2; extreme race condition = abort |

## Idempotence

Re-run on the same clean state is a no-op:
- Workspace branch absent → warning + skip
- `.icm-main/` absent → skip
- Project_root on main → skip checkout
- No orphan subagents → skip
- `git worktree prune` is always safe

Running 2× in a row produces the same final state, exit code 0 on both.

## Recovery wizard integration

New detector `STALE_ICM_MAIN_AFTER_CLOSE` (v3.7.2):
- Trigger: `workspaces/.index.md` has zero active workspaces + `.icm-main/` present.
- Plan A: offers to invoke `icm-cleanup.py` interactively.
- Useful for pre-v3.7.2 projects that finished exit A/C before the auto-trigger.

## Manual human use

After exit A/C, if human said `[n]` in the menu (skip cleanup) and wants to run it later:

```bash
python <skill-dir>/scripts/icm-cleanup.py \
    --project-root <project-root> \
    --workspace <NNN-slug> \
    --dry-run     # check first
```

Confirm output, then run without `--dry-run` to execute.
